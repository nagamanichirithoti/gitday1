What is Git?

Git is a version control system that tracks changes in source code during software development. It enables multiple developers to work together and track revisions.

What is GitHub?

GitHub is a web-based platform that hosts Git repositories, providing additional features for collaboration like pull requests, issue tracking, and more. It allows developers to work on projects, share code, and manage versions.

Basic Git Workflow

Setting up a repository: Create a local repository to manage files and track changes.

Making changes: Modify code or add new files.

Staging changes: Mark changes to be committed.

Committing changes: Save a snapshot of the current changes.

Pushing to a remote repository: Upload changes to GitHub or another Git server.

Collaborating: Fetch and merge changes from others.

Branching: Work on new features or fixes in isolated environments.

Merging: Combine branches into the main branch.

Essential Git Commands

1. Initial Setup Commands
git init
Initializes a new Git repository. Run this in the root directory of your project.
bash
Copy code
git init
git clone [URL]
Copies an existing repository from GitHub to your local machine.
bash
Copy code
git clone https://github.com/user/repo.git
git remote add origin [URL]
Connects your local repository to a remote one (like on GitHub).
bash
Copy code
git remote add origin https://github.com/user/repo.git
2. Making Changes
git status
Shows the status of changes in your working directory. It's useful to check which files have been modified or staged.
bash
Copy code
git status
git add [file_name] or git add .
Stages a file or all files (.) to be committed.
bash
Copy code
git add file.txt
git add .
git commit -m "[message]"
Commits staged changes with a descriptive message.
bash
Copy code
git commit -m "Add feature X"
3. Working with Remote Repositories
git push origin [branch_name]
Pushes your changes to the remote repository. For example, to push to the main branch:
bash
Copy code
git push origin main
git pull origin [branch_name]
Fetches changes from the remote branch and merges them into the current branch.
bash
Copy code
git pull origin main
git fetch
Downloads updates from the remote repository without merging.
bash
Copy code
git fetch
4. Branching and Merging
git branch [branch_name]
Creates a new branch. A branch is an isolated environment to work on specific changes.
bash
Copy code
git branch feature-branch
git checkout [branch_name]
Switches to the specified branch.
bash
Copy code
git checkout feature-branch
git merge [branch_name]
Merges the specified branch into the current branch.
bash
Copy code
git merge feature-branch
git branch -d [branch_name]
Deletes a branch.
bash
Copy code
git branch -d feature-branch
5. Undoing Changes
git reset [file_name]
Unstages a file, moving it from the staging area back to modified.
bash
Copy code
git reset file.txt
git revert [commit_id]
Creates a new commit that undoes the changes of a specific commit.
bash
Copy code
git revert a1b2c3d
